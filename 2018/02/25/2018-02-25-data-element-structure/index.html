数据，数据元素，数据结构

  • Data Structures
Programming

数据

数据（data）是对客观事物的量化，是计算机识别，存储和处理的对象
信息（information）经过收集和处理后的数据
数据和信息关系：数据是信息的载体，信息是数据的含义

数据元素

数据元素（data element）是数据的基本单位，由数据项组成，数据项是具有独立含义的最小标识单位

数据结构

数据结构（data structure）是计算机存储和组织数据的方式，具体指同一类数据元素中，各元素之间的相互关系，包括三个组成成分，即数据的逻辑结构，数据的存储结构和数据的运算

逻辑结构

逻辑结构（logic structure）是指数据元素之间的逻辑关系，独立于数据在计算机的存储方式，数据的逻辑结构可以看作是从具体问题抽象出来的数学模型

  • 线性结构 ：即数据元素之间是线性关系，除了首尾节点，每个只有一个前驱节点和后继节点
  • 非线性结构 ：即数据元素之间是非线性关系

存储结构

存储结构（physical structure）是数据元素之间的存储关系，依赖于数据在计算机的存储方式

  • 顺序存储结构：在计算机中用一组连续的存储单元依次数据元素（优点：随机存储，查找速度快，缺点：增删需要移动数据元素）
  • 链式存储结构：在计算机中用一组任意的存储单元存储数据元素，通过指针表示数据元素之间的逻辑关系（优点：增删灵活，缺点：存储空间需求大，查找慢）
  • 索引存储结构：在链式存储的基础上，通过附加的索引表标识数据元素存储地址（优点：查找速度快，缺点：占用更多存储空间）
  • 散列存储结构：通过散列技术保证查找速度的同时压缩存储空间，通过数据元素的散列值决定存储地址

运算

数据的运算是定义在数据的逻辑结构上的运算集合，比如增删改查。这些运算都是抽象的操作，只有确定了存储结构才能确定具体的操作
